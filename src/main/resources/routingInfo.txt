POST /auth/registration - регистрация нового пользователя с получением токена

тело JSON для регистрации такое:

    {
        "firstName": "Stepan",
        "lastName": "Fedosov",
        "phoneNumber": "+79996789831",
        "email": "zollidan@yandex.ru",
        "password": "1234",
        "newsletter": 1
    }

    после реги сразу получаешь токен для авторизации этого человека 

POST /auth/login - логин пользователя 

тело JSON для логина такое 

    {
        "email": "zollidan@yandex.ru",
        "password": "1234"
    }

    и тебе возвращается токен в JSON "jwt-token": "fvwer3rb3rbevjdnjvnjdn..."
    токен дается на 60 минут тут будет описано как продлить ................ 

    короче для того чтобы делать запросы на защищенные ссылки нужно при запросе приложить токен токен пользователя в запрос

    конкретно такая строка нужна 

    "Authorization": "Bearer egnjkenvjnenjnjwrnvjkwvjkn"

    после Bearer есть пробел он тоже считается


все остальные запросы начинаются с /api/

типо /api/item/{id}

/item/{id} - ищешь товар по айди в бд

/allItems - все товары вообще из всех категорий

/{categoryId}/allItems - в переменной номер категории товара и ты получаешь все товары категории

на этом разрешенные запросы кончаются все остальные запросы требуют входа в попку под видом админа или юзера
по стандарту дается юзер админ функции для него недоступны а для админа все доспно соответственно

функции юзера обычного далее:
/person/info - получение информации о пользователи в сессии

/orderhistory - история заказов пользователя

/sluts - сервис заказа шалав(пока не делал)
